# Changes Log

This folder contains documentation of major updates, features, and architectural changes to the WorkoutLog codebase.

## Purpose

- **Historical Record:** Track what was changed and why
- **Onboarding:** Help new contributors understand the codebase evolution
- **Decision Documentation:** Preserve justifications for architectural decisions
- **Release Notes Source:** Provide material for user-facing changelogs

## File Naming Convention

Files follow the format: `YYYY-MM-DD-description.md`

Examples:
- `2024-12-30-codebase-refactoring.md`
- `2024-12-25-timer-notifications.md`
- `2025-01-15-dark-mode-support.md`

## What to Document

### Major Changes (Always Document)
- Architectural refactoring
- New feature implementations
- Breaking changes
- Dependency upgrades with impact
- Performance optimizations

### Minor Changes (Optional)
- Bug fixes (unless significant)
- Small UI tweaks
- Dependency patches

## Template

```markdown
# [Title] - [Date]

## Overview
Brief summary of what was done and why.

## Changes Made
### 1. [Category]
**Files Added/Modified:**
- `path/to/file.ts`

**Justification:**
Why this change was made.

## Impact Summary
Tables or metrics showing before/after.

## Migration Notes
Any changes developers need to be aware of.
```
